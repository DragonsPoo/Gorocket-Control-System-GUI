# GoRocket 지상 제어·모니터링 시스템 운용 매뉴얼

이 프로그램은 지상에서 액체로켓 엔진을 안전하게 테스트하기 위해 개발된 제어 및 모니터링 시스템입니다. 아두이노 메가 2560과 유선으로 연결하여 밸브 제어, 센서 모니터링, 자동화된 시퀀스 실행을 수행하며, 무엇보다 안전을 최우선으로 설계되었습니다.

## 1. 시스템 이해하기

### 이 시스템이 하는 일

이 프로그램은 마치 항공기의 조종석처럼, 복잡하고 위험한 로켓 테스트 하드웨어들을 한 곳에서 안전하게 조작할 수 있게 해줍니다. 컴퓨터 화면에서 버튼을 클릭하면 실제 밸브가 열리고 닫히며, 실시간으로 압력과 온도를 모니터링하여 위험한 상황을 미리 감지할 수 있습니다.

**핵심 기능들:**
- **실시간 모니터링**: 압력, 온도, 유량 센서의 데이터를 0.1초마다 확인
- **밸브 제어**: 7개의 밸브를 개별적으로 또는 자동 시퀀스로 제어
- **자동화 테스트**: 복잡한 테스트 절차를 미리 정의해두고 자동으로 실행
- **비상 안전 시스템**: 위험 상황에서 즉시 모든 시스템을 안전한 상태로 전환
- **데이터 기록**: 모든 센서 데이터와 조작 기록을 파일로 저장

### 시스템 구성 요소

**컴퓨터 (GUI 프로그램)**
- Windows 10/11에서 실행되는 Electron 기반 데스크톱 애플리케이션
- 사용자가 보는 화면과 모든 제어 인터페이스 제공
- 센서 데이터를 실시간 그래프로 표시
- 테스트 로그를 CSV 파일로 자동 저장

**아두이노 메가 2560 (하드웨어 제어기)**
- 실제 센서들로부터 데이터를 읽어옴
- 서보 모터를 통해 밸브들을 제어
- 안전 한계값을 상시 감시하여 비상시 자동 차단
- USB 시리얼 통신으로 컴퓨터와 데이터 주고받음

**센서 하드웨어**
- 압력 센서 4개: 연료탱크, 산화제탱크, 연료라인, 산화제라인 압력 측정
- 온도 센서 2개: 연소실과 노즐 출구 온도 측정 (최대 800°C)
- 유량 센서 2개: 연료와 산화제의 실시간 유량 측정
- 리밋스위치 4개: 밸브가 실제로 열렸는지/닫혔는지 확인

## 2. 설치 및 첫 실행

### 필수 프로그램 설치

이 시스템을 사용하기 위해서는 먼저 컴퓨터에 Node.js가 설치되어 있어야 합니다.

1. **Node.js 설치**: https://nodejs.org 에서 LTS 버전 다운로드 후 설치
2. **아두이노 IDE 설치**: https://arduino.cc 에서 다운로드 (아두이노 펌웨어 업로드용)

### 프로그램 설치 및 실행

Windows PowerShell을 관리자 권한으로 열고 다음 명령어들을 순서대로 실행하세요:

```powershell
# 1. 프로젝트 폴더로 이동
cd "C:\Users\[사용자명]\Gorocket-Control-System-GUI"

# 2. 필요한 라이브러리들을 설치
npm ci

# 3. 프로그램에 오류가 없는지 확인
npm run typecheck

# 4. 프로그램을 빌드
npm run build

# 5. 개발 모드로 실행
npm run dev
```

정상적으로 실행되면 자동으로 웹 브라우저가 열리면서 제어 화면이 나타납니다.

### 아두이노 펌웨어 설치

1. 아두이노 IDE를 열고 `arduino_mega_code.ino` 파일을 엽니다
2. 보드를 "Arduino Mega 2560"으로 선택합니다
3. 올바른 COM 포트를 선택합니다
4. 업로드 버튼을 클릭하여 펌웨어를 설치합니다

## 3. 안전 시스템 이해하기

### 3단계 안전 보호 체계

이 시스템은 3중 안전 보호 체계로 설계되어 있습니다. 각 단계는 독립적으로 작동하여 하나가 실패해도 다른 단계에서 안전을 보장합니다.

**1단계: 소프트웨어 감시 (0.1초마다 확인)**
- 컴퓨터 프로그램이 모든 센서 값을 실시간으로 확인
- 압력이나 온도가 안전 한계를 넘으면 즉시 경고 표시
- 사용자가 화면에서 확인하고 수동으로 대응 가능

**2단계: 아두이노 하드웨어 감시 (0.1초마다 확인)**
- 아두이노가 독립적으로 모든 센서를 감시
- 위험 상황 감지 시 컴퓨터의 명령 없이도 자동으로 비상 차단
- 통신이 끊어져도 계속 작동하는 완전 독립형 안전 시스템

**3단계: 물리적 안전 장치**
- 수동 비상 정지 스위치 (모든 전원 차단)
- 압력 릴리프 밸브 (과압 시 자동 방출)
- 물리적 차단막과 안전 거리 확보

### ARM/DISARM 시스템

이 시스템에는 실수로 위험한 조작을 하는 것을 방지하기 위해 ARM/DISARM 기능이 있습니다.

**DISARM 상태 (기본값)**
- 모든 밸브 제어가 비활성화됨
- 센서 모니터링만 가능
- 안전한 상태에서 시스템 점검 가능

**ARM 상태**
- 모든 제어 기능이 활성화됨
- 밸브 제어 및 자동 시퀀스 실행 가능
- 실제 테스트 실행 시에만 ARM 상태로 전환

ARM 상태로 전환하려면 화면 상단의 "ARM" 버튼을 클릭해야 합니다. 비상 상황이나 오류 발생 시에는 자동으로 DISARM 상태로 돌아갑니다.

### 비상 정지 절차

**소프트웨어 비상 정지**
- 키보드 단축키: `Ctrl + Shift + E`
- 화면의 빨간색 "EMERGENCY STOP" 버튼 클릭
- 실행 중인 모든 시퀀스 즉시 중단
- 모든 주요 밸브 자동 차단 (0.5초 이내)

**하드웨어 비상 정지**
- 아두이노가 자동으로 감지하는 상황:
  - 압력이 설정 한계 초과 (1000 PSI)
  - 압력 상승률 초과 (50 PSI/초)
  - 컴퓨터와 통신 두절 (3초 이상)
- 감지 즉시 모든 밸브를 안전 위치로 이동

## 4. 일상적인 운용 절차

### 매일 테스트 전 점검 사항 (소요시간: 15분)

**1. 하드웨어 육안 점검**
- 모든 케이블 연결 상태 확인
- 센서 마운팅 상태 점검
- 밸브 및 배관 누설 여부 확인
- 비상 정지 스위치 동작 테스트

**2. 소프트웨어 연결 확인**
- 프로그램 실행 및 아두이노 연결
- 모든 센서 데이터가 정상적으로 수신되는지 확인
- 압력 센서: 대기압 상태에서 0 PSI 근처 값
- 온도 센서: 실내 온도 (20-30°C) 정상 표시
- 유량 센서: 0 L/min 표시

**3. 밸브 동작 테스트**
- ARM 상태로 전환
- 각 밸브를 개별적으로 열고 닫기 테스트
- 리밋스위치가 정확한 위치를 표시하는지 확인
- 모든 밸브를 닫힌 상태로 복원
- DISARM 상태로 전환

### 테스트 실행 절차

**1. 사전 준비 (소요시간: 10분)**
- 안전 거리 확보 (최소 50m)
- 소화 장비 준비 및 위치 확인
- 통신 장비 테스트 (무전기 등)
- 기상 조건 확인 (바람, 습도)

**2. 시스템 시작**
- 프로그램 실행 및 연결 확인
- 로깅 시작 (화면 상단의 "Start Logging" 버튼)
- ARM 상태로 전환

**3. 수동 테스트 실행**
- 원하는 밸브를 화면에서 클릭하여 제어
- 실시간 센서 데이터를 지속적으로 모니터링
- 이상 상황 발견 시 즉시 비상 정지

**4. 자동 시퀀스 테스트 실행**
- 미리 정의된 시퀀스 선택
- "실행" 버튼 클릭으로 자동 테스트 시작
- 진행 상황을 실시간으로 모니터링
- 필요시 "중단" 버튼으로 즉시 정지

**5. 테스트 종료**
- 모든 밸브가 안전한 위치에 있는지 확인
- 압력이 완전히 방출되었는지 확인 (0 PSI)
- 로깅 중지 및 데이터 파일 저장
- DISARM 상태로 전환

## 5. 센서 시스템 이해하기

### 압력 센서 (가장 중요한 안전 센서)

**측정 원리**
압력 트랜스듀서가 물리적인 압력을 전기 신호로 변환합니다. 0-5V 전압이 0-100 PSI 압력에 대응됩니다. 아두이노가 이 전압을 0.1초마다 읽어서 실제 압력값으로 계산합니다.

**4개 센서의 역할**
- **연료탱크 압력**: 에탄올이 저장된 탱크의 가압 상태 감시
- **산화제탱크 압력**: 아산화질소가 저장된 탱크의 압력 감시
- **연료라인 압력**: 연소실로 가는 연료 라인의 실시간 압력
- **산화제라인 압력**: 연소실로 가는 산화제 라인의 실시간 압력

**정상 운용 범위**
- 일반 운용: 0-850 PSI (정상)
- 경보 수준: 850 PSI 이상 (소프트웨어 경고)
- 트립 수준: 1000 PSI 이상 (하드웨어 자동 차단)

**위험 상황 감지**
압력이 안전 한계를 넘으면 시스템이 자동으로:
1. 화면에 빨간색 경고 표시
2. 경고음 발생 (부저)
3. 주요 밸브 자동 차단
4. 압력 방출 밸브 자동 개방

### 온도 센서

**측정 원리**
K-타입 열전대가 온도를 전압으로 변환하고, MAX6675 칩이 이를 디지털 신호로 변환합니다. 최대 800°C까지 측정 가능하며, 0.25°C 정확도를 가집니다.

**2개 센서의 역할**
- **연소실 온도**: 실제 연소가 일어나는 챔버 내부 온도
- **노즐 출구 온도**: 고온 가스가 분사되는 노즐 끝 온도

**정상 운용 범위**
- 대기 상태: 20-40°C
- 테스트 중: 200-600°C (정상), 600-800°C (주의), 800°C 초과 (위험)

### 유량 센서

**측정 원리**
터빈식 유량계가 액체 흐름에 의해 회전하며, 홀 센서가 이 회전을 전기 펄스로 변환합니다. 아두이노가 펄스 개수를 세어서 실제 유량(L/min)으로 계산합니다.

**2개 센서의 역할**
- **연료 유량**: 에탄올의 실시간 소모량 측정
- **산화제 유량**: 아산화질소의 실시간 소모량 측정

**혼합비 모니터링**
두 유량 센서의 비율을 실시간으로 계산하여 최적의 연소 조건을 유지합니다. 일반적으로 산화제 대 연료 비율이 2:1에서 3:1 사이가 이상적입니다.

## 6. 밸브 제어 시스템

### 밸브 작동 원리

**서보 모터 제어**
각 밸브는 정밀 서보 모터로 제어됩니다. 아두이노가 PWM(펄스폭변조) 신호를 보내면 서보가 정확한 각도로 회전하여 밸브를 열거나 닫습니다.

**실제 서보 각도 설정:**
- 각 밸브마다 고유한 열림/닫힘 각도를 가짐
- 예시: V1은 7도(열림)/103도(닫힘), V2는 25도(열림)/121도(닫힘)
- 이동 시간: 약 0.5초 + 안정화 시간 0.5초 = 총 1초

**위치 확인 시스템**
각 밸브에는 리밋스위치가 2개씩 설치되어 있어서:
- 밸브가 완전히 열렸는지 확인
- 밸브가 완전히 닫혔는지 확인
- 서보 모터 고장 시에도 실제 밸브 상태를 정확히 알 수 있음

### 7개 밸브의 역할 (실제 시스템 기준)

**V1: Ethanol Purge Line (에탄올 퍼지 라인)**
- 테스트 후 연료 라인 내부 청소
- 잔여 에탄올 제거 및 질소 퍼지
- 안전을 위해 테스트 전후에 반드시 사용

**V2: Main Pressurization (메인 가압)**
- 시스템 전체 가압 제어
- 테스트 시 필요한 압력 공급
- 압력 조절의 핵심 밸브

**V3: Ethanol Fill Line (에탄올 충전 라인)**
- 연료 탱크로의 에탄올 공급 라인
- 연료 충전 및 공급 제어
- 테스트 준비 단계에서 사용

**V4: N2O Main Supply (아산화질소 주 공급) - 가장 중요**
- 연소실로 들어가는 산화제 공급의 핵심
- 테스트의 시작과 끝을 결정하는 주요 밸브
- 비상시 가장 먼저 차단되는 밸브

**V5: Ethanol Main Supply (에탄올 주 공급) - 가장 중요**
- 연소실로 들어가는 연료 공급의 핵심
- N2O 주 공급 밸브와 함께 연소를 제어
- 비상시 두 번째로 차단되는 밸브

**V6: System Vent 1 (시스템 벤트 1)**
- 시스템 압력 방출용 벤트
- 비상시 압력 해제
- 안전 절차에서 중요한 역할

**V7: System Vent 2 (시스템 벤트 2)**
- 시스템 압력 방출용 보조 벤트
- 빠른 압력 해제를 위한 추가 경로
- 비상 상황에서 압력 분산 역할

### 안전한 밸브 조작 순서

**테스트 시작 순서**
1. 메인 가압 (V2) 먼저 - 시스템 압력 형성
2. 압력이 충분히 올라가면 주 공급 밸브들 (V4-N2O, V5-에탄올) - 연소 시작
3. 설정된 시간 후 주 공급 밸브 차단 - 연소 종료
4. 에탄올 퍼지 라인 (V1) 개방 - 연료 라인 청소

**비상 차단 순서**
1. N2O 주 공급 (V4) 즉시 차단 - 산화제 차단으로 연소 중단
2. 에탄올 주 공급 (V5) 차단 - 연료 차단으로 완전 정지
3. 시스템 벤트들 (V6, V7) 개방 - 압력 방출
4. 에탄올 퍼지 라인 (V1) 개방 - 연료 라인 청소

## 7. 자동 시퀀스 시스템

### 시퀀스란 무엇인가

자동 시퀀스는 복잡한 테스트 절차를 미리 정의해 두고, 컴퓨터가 정확한 타이밍에 맞춰 자동으로 실행하는 기능입니다. 사람이 수동으로 하기에는 너무 빠르거나 정확한 타이밍이 필요한 작업들을 안전하고 정확하게 수행할 수 있습니다.

**장점**
- 사람의 실수 방지
- 정확한 타이밍 보장 (밀리초 단위)
- 복잡한 절차의 자동화
- 재현 가능한 테스트 조건

**안전 기능**
- 각 단계마다 안전 확인
- 이상 상황 감지 시 자동 중단
- 실시간 수동 개입 가능

### 시퀀스 파일 이해하기

시퀀스는 `sequences.json` 파일에 정의되어 있습니다. 각 시퀀스는 시간 순서대로 배열된 단계들로 구성됩니다.

**기본 구조**
```
시퀀스 이름: "Cold Flow Test"
설명: "연료만 사용한 냉간 유동 테스트"
예상 소요시간: 30초

단계들:
T+0초: 연료 가압 시작 (V5 열기)
T+5초: 압력 50PSI 대기
T+10초: 연료 주 밸브 열기 (V4 열기)
T+15초: 연료 주 밸브 닫기 (V4 닫기)
T+20초: 연료 가압 중단 (V5 닫기)
T+25초: 퍼지 시작 (V0 열기)
T+30초: 퍼지 종료 (V0 닫기)
```

### 주요 시퀀스 종류

**1. Cold Flow Test (냉간 유동 테스트)**
- 연료만 사용하여 시스템 동작 확인
- 실제 연소 없이 유량 패턴 확인
- 안전성이 높아 초보자 연습용으로 적합
- 소요시간: 약 30초

**2. Hot Fire Test (열간 연소 테스트)**
- 연료와 산화제를 모두 사용한 실제 연소 테스트
- 가장 위험하지만 실제 엔진 성능 확인 가능
- 숙련된 운용자만 실행 권장
- 소요시간: 약 10-15초

**3. Pressure Test (압력 테스트)**
- 시스템의 압력 한계 확인
- 누설 지점 탐지
- 연소 없이 압력만 사용
- 소요시간: 약 60초

**4. Emergency Shutdown (비상 차단 테스트)**
- 비상 상황 대응 절차 연습
- 모든 안전 시스템 동작 확인
- 정기적으로 실행하여 안전성 확보
- 소요시간: 약 5초

### 시퀀스 실행 과정

**1. 시퀀스 선택**
- 화면 오른쪽 "자동 시퀀스 제어" 패널에서 원하는 시퀀스 선택
- 시퀀스 이름을 클릭하면 상세 단계들을 미리 볼 수 있음
- 예상 소요시간과 위험도 확인

**2. 안전 확인**
- 시스템이 ARM 상태인지 확인
- 모든 센서가 정상 범위에 있는지 확인
- 비상 정지 준비 상태 확인

**3. 실행 시작**
- "실행" 버튼 클릭
- 시스템이 자동으로 사전 안전 검사 수행
- 문제가 없으면 첫 번째 단계 시작

**4. 실행 중 모니터링**
- 화면에 현재 진행 상황과 다음 단계 표시
- 실시간 센서 데이터 지속 모니터링
- 이상 상황 발견 시 즉시 "중단" 버튼 클릭

**5. 완료 또는 중단**
- 정상 완료: 모든 단계가 순서대로 실행된 후 자동 종료
- 수동 중단: "중단" 버튼으로 즉시 정지 후 안전 절차 실행
- 자동 중단: 안전 한계 초과 시 시스템이 자동으로 중단

## 8. 사용자 인터페이스 안내

### 메인 화면 구성

화면은 크게 5개 영역으로 나뉘어 있습니다:

**상단 헤더**
- 연결 상태 표시 (녹색: 연결됨, 빨간색: 연결 끊김)
- ARM/DISARM 버튼 (시스템 활성화/비활성화)
- 로깅 시작/중지 버튼
- 비상 정지 버튼 (큰 빨간 버튼)

**센서 모니터링 패널 (화면 위쪽)**
- 8개 센서의 실시간 데이터를 한 눈에 표시
- 정상: 녹색, 주의: 노란색, 위험: 빨간색으로 색상 구분
- 각 센서의 현재값과 안전 한계값 표시

**밸브 제어 패널 (화면 왼쪽 아래)**
- 7개 밸브를 개별적으로 제어하는 버튼들
- 각 밸브의 현재 상태 (열림/닫힘) 실시간 표시
- ARM 상태일 때만 버튼 활성화

**자동 시퀀스 패널 (화면 오른쪽 위)**
- 사용 가능한 시퀀스 목록
- 현재 실행 중인 시퀀스의 진행률 표시
- 시퀀스 실행/중단 버튼

**데이터 차트 패널 (화면 왼쪽 아래)**
- 센서 데이터의 실시간 그래프
- 최근 1분간 데이터를 자동으로 표시
- 특정 센서만 선택해서 보기 가능

**로그 패널 (화면 오른쪽 아래)**
- 모든 시스템 동작과 메시지를 시간순으로 기록
- 오류나 경고 메시지는 색상으로 구분
- 자동 스크롤 기능으로 최신 메시지 항상 표시

### 키보드 단축키

자주 사용하는 기능들을 키보드로 빠르게 실행할 수 있습니다:

- `Ctrl + Shift + E`: 비상 정지
- `Ctrl + Shift + A`: ARM/DISARM 토글
- `Ctrl + =`: 화면 확대
- `Ctrl + -`: 화면 축소
- `Ctrl + 0`: 화면 크기 초기화

### 색상 코드 이해하기

시스템 전체에서 일관된 색상 코드를 사용합니다:

- **녹색**: 정상 상태, 안전 범위
- **노란색**: 주의 상태, 모니터링 강화 필요
- **빨간색**: 위험 상태, 즉시 조치 필요
- **회색**: 비활성 상태, 사용 불가
- **파란색**: 정보 표시, 일반 메시지

## 9. 데이터 로깅과 분석

### 로깅 시스템

모든 테스트 데이터는 자동으로 기록되어 나중에 분석할 수 있습니다.

**로그 파일 위치**
- Windows: `C:\Users\[사용자명]\Documents\rocket-logs\`
- 각 세션마다 별도 폴더 생성 (예: `session-20231215-143022`)

**저장되는 데이터**
- 모든 센서 데이터 (0.1초 간격)
- 밸브 조작 기록 (정확한 시간 포함)
- 시퀀스 실행 로그
- 오류 및 경고 메시지
- 시스템 설정 스냅샷

**CSV 파일 형식**
```
타임스탬프, 압력1, 압력2, 압력3, 압력4, 온도1, 온도2, 유량1, 유량2, 밸브상태
2023-12-15T14:30:22.123Z, 45.2, 48.1, 52.3, 49.7, 25.4, 28.1, 0.0, 0.0, V1:CLOSED V2:OPEN
```

### 데이터 분석 방법

**Excel에서 분석**
1. CSV 파일을 Excel로 열기
2. 타임스탬프를 시간 축으로 하여 그래프 생성
3. 압력 변화 패턴 분석
4. 온도 상승 곡선 확인

**전문 분석 도구**
- Python pandas: 대량 데이터 처리
- MATLAB: 고급 신호 분석
- LabVIEW: 실시간 데이터 처리

**주요 분석 지표**
- 최대 압력 도달 시간
- 압력 상승률 (PSI/초)
- 온도 변화 패턴
- 연소 지속 시간
- 연료 소모량

## 10. 문제 해결 가이드

### 자주 발생하는 문제들

**문제: 아두이노에 연결이 안 됨**
- 원인: USB 케이블 불량, 드라이버 문제, 포트 충돌
- 해결방법:
  1. USB 케이블을 다른 것으로 교체해보기
  2. 장치 관리자에서 COM 포트 확인
  3. 아두이노 IDE에서 연결 테스트
  4. 컴퓨터 재시작 후 재시도

**문제: 센서 데이터가 이상함**
- 원인: 센서 불량, 배선 문제, 보정 오류
- 해결방법:
  1. 센서 전원 및 신호선 점검
  2. 다른 센서와 비교하여 이상값 확인
  3. 센서 교정 절차 다시 실행
  4. 필요시 센서 교체

**문제: 밸브가 동작하지 않음**
- 원인: 서보 모터 불량, 전원 부족, 기계적 걸림
- 해결방법:
  1. 서보 모터 전원 확인 (5V)
  2. 기계적 장애물 제거
  3. 수동으로 밸브 동작 확인
  4. 서보 모터 교체

**문제: 시퀀스가 중간에 멈춤**
- 원인: 센서 값 이상, 안전 한계 초과, 통신 오류
- 해결방법:
  1. 로그 패널에서 오류 메시지 확인
  2. 모든 센서가 정상 범위에 있는지 점검
  3. 시스템 재시작 후 다시 시도
  4. 더 간단한 시퀀스로 먼저 테스트

### 비상 상황 대응

**압력 이상 상승**
1. 즉시 비상 정지 버튼 클릭
2. 수동으로 벤트 밸브들 개방
3. 압력이 0에 가까워질 때까지 대기
4. 원인 파악 후 안전 확인 되면 재시작

**온도 이상 상승**
1. 비상 정지 및 모든 연료 공급 차단
2. 냉각수 분사 (준비된 경우)
3. 안전 거리 확보 및 소방서 신고 고려
4. 온도가 정상으로 돌아올 때까지 대기

**통신 두절**
1. USB 케이블 연결 상태 확인
2. 프로그램 재시작
3. 아두이노 리셋 버튼 누르기
4. 시스템 전체 재시작

**비상 상황 시 절대 금지사항**
- 비상 상황에서 시스템 강제 재시작
- 압력이 남아있는 상태에서 하드웨어 조작
- 원인 불명 시 즉시 재테스트
- 안전 장비 없이 가까이 접근

## 11. 정기 점검 및 유지보수

### 일일 점검 항목

**시작 전 점검 (매일)**
- 모든 케이블 연결 상태
- 센서 마운팅 고정 상태
- 밸브 동작 부드러움
- 소프트웨어 연결 및 데이터 수신

**종료 후 점검 (매일)**
- 모든 밸브 닫힘 확인
- 시스템 내부 압력 완전 방출
- 로그 파일 저장 확인
- 하드웨어 외관 손상 여부

### 주간 점검 항목

**센서 정확도 확인**
- 압력 센서: 알려진 압력원과 비교
- 온도 센서: 정확한 온도계와 비교
- 유량 센서: 정확한 유량원과 비교

**밸브 성능 확인**
- 개폐 속도 측정 (정상: 0.5-1초)
- 리밋스위치 정확도 확인
- 서보 모터 소음 및 진동 점검

### 월간 점검 항목

**전체 시스템 교정**
- 모든 센서 교정 절차 실행
- 안전 한계값 재확인
- 비상 정지 시스템 동작 테스트

**소프트웨어 업데이트**
- 최신 버전 확인 및 업데이트
- 시퀀스 파일 백업 및 점검
- 로그 파일 정리 및 보관

### 보관 및 관리

**장기 보관 시 (1개월 이상 미사용)**
- 모든 연료 및 산화제 완전 제거
- 시스템 내부 질소 퍼지
- 전자 부품 방습 처리
- 정기적인 동작 테스트 (월 1회)

**데이터 백업**
- 중요한 테스트 데이터는 여러 곳에 백업
- 클라우드 저장소 활용 권장
- 시퀀스 파일 및 설정 파일 별도 보관

## 12. 고급 활용법

### 커스텀 시퀀스 작성

더 복잡하거나 특별한 테스트를 위해 새로운 시퀀스를 만들 수 있습니다.

**시퀀스 설계 원칙**
- 안전 확인 단계를 충분히 포함
- 각 단계 사이에 적절한 시간 간격 확보
- 비상 상황에 대한 대응 절차 포함
- 테스트 목적에 맞는 센서 모니터링

**기본 템플릿**
```
1. 시작 안전 확인
2. 시스템 가압
3. 압력 안정화 대기
4. 주 테스트 실행
5. 정상 종료 절차
6. 안전 확인 및 정리
```

### 원격 모니터링

네트워크를 통해 원격에서 시스템을 모니터링할 수 있습니다.

**설정 방법**
1. 로컬 네트워크에서 웹 브라우저로 접속
2. 실시간 센서 데이터 원격 확인
3. 비상 상황 시 원격 비상 정지 가능

**보안 주의사항**
- 반드시 로컬 네트워크에서만 사용
- 인터넷 연결 시 보안 위험 존재
